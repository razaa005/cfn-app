Information Syndication API

Introduction

Welcome to the BBC's Information Syndication API, this allows authorised users to be able to take specific content from the BBC for syndication onto their websites or proprietary platforms.

If you are attempting to access this API then you should have a relationship with the BBC, BBC World Service or BBC Global News Ltd. If you have any questions that are not covered by this documentation then please contact your Account Manager who will be happy to help.


API Keys

An API Key is a long string of characters that must be included in the Query String parameters of your request to the API. Without a valid API key you will not be granted access to the content.

Example

Requesting an API Key

An API Key should have been provided to you by your Account Manager when they setup your contract for syndication. Please contact them if this is not the case.

Once you have this key you must keep it secure to prevent other people from gaining access to our API by using it. Do not make calls using this key directly from the client-side of your website.


Request Structure

URL: https://information-syndication.api.bbc.com/articles

Parameter Name

Type

Description

api_key

Query String

The security key required for access (see API Keys section).

feed

Query String

The name of the BBC feed you wish to syndicate (see What is a Feed?)

Accept

HTTP Header

Pass “application/rss+xml” to receive the RSS 2.0 formatted response. If you require another format for your integration, then this will be given in your instructions. Defaults to RSS 2.0 if not provided.

accept_override

Query String

Pass “rss2” to receive the RSS 2.0 formatted response. This should only be used when you cannot set the Accept header. If you require another format for your integration then this will be given in your instructions. Defaults to Accept header value if not provided.

mixins

Query String

This allows a comma separated list of values which will add certain things into the feed. For example, “summary” will mean that you get the description of the article. By default only the headline, last publish Date, URL & GUID are included in the feed. Please see table below for different values and what they do.

Note, you can also pass each of them as an individual “mixin” query string parameter rather than a comma separated list if you prefer.

number_of_items

Query String

The maximum number of items to return in the feed from 1 — 25.

sort

Query String

Controls the sort order of the returned feed.

“date_asc”: sorts by last publish date of the article, ascending

“date_desc”: sorts by last publish date of the article, descending

Defaults to no sort being applied.

clip_format

Query String

If the “clips” mixin is passed, this determines what format of clip to use:

“link”: results in the URL to an MP3 or MP4 file that you can download to serve to users or include directly in the user's browser

“embedded_player”: includes an <iframe> to the BBC's embeddable media player where video or audio clips appear in articles.

Defaults to “link”.

For further information see here: for Text Article feeds or for Audio/Video only feeds

thumbnail_image_format

Query String

Legacy support only: This should only be used if you cannot parse the standard thumbnail image format.

Controls the output format for thumbnail images.

“image_only”: changes the thumbnail image format to provide only the image url, width and height. If this is used then only BBC-owned thumbnail images without copyright information, title, or description will be provided. The default BBC site logo will be used to replace all non-BBC owned thumbnail images.

Exclude this query parameter to receive the standard thumbnail images format.

To see examples of the thumbnail image output see here

social_format

Query String

Determines how social media content (e.g. YouTube, Twitter, Instagram, etc...) appears within articles:

“link”: means that a link to the social media content appears as an anchor tag. (e.g. <a href>)

“embedded”: means that the HTML of the content is delivered from the social media service as per their documentation

Defaults to “link”.

twitter_format

Query String

Deprecated: Please use “social_format” instead.

Determines how tweets appear within articles:

“link”: means that a link to the tweet appears as an anchor tag. (e.g. <a href>)

“embedded”: means that the HTML of the tweet is delivered from Twitter inside a <blockquote> as per their documentation

Defaults to “link”.

youtube_format

Query String

Deprecated: Please use “social_format” instead.

Determines how YouTube videos appear within articles:

“link”: means that a link to the video appears as an anchor tag. (e.g. <a href>)

“embedded”: means that the YouTube embedded player <iframe> is inserted

Defaults to “link”.

instagram_format

Query String

Deprecated: Please use “social_format” instead.

Determines how Instagram photos appear within articles:

“link”: means that a link to the post appears as an anchor tag. (e.g. <a href>)

“embedded”: means that the HTML of the post is delivered from Instagram inside a <blockquote> as per their documentation

Defaults to “link”.

page_view_tracking

Query String

Selects the format of the page view tracking to be inserted into article body content, either “image” or “javascript”. Defaults to “image” if not passed, requires that page view tracking be enabled for you.

Mixin Values

These are values to be passed into the “mixins” query string parameter. See the example RSS 2.0 response below to how all of these elements appear in the final feed.

Value Name

Description

summary

Returns the summary description of each article in the feed in <description> node (RSS 2.0).

thumbnail_images

Returns the images used to link to the article from index pages in <media:thumbnail> nodes contained within <media:content> nodes (RSS 2.0). The <media:content> tags also contain copyright and alt text information.

Note that there may be more than one thumbnail image per article, we have no preference over which you use in this case, although highest resolution might be a good guide.

body

Returns the text contents of each article in the feed in <content:encoded> node (RSS 2.0).

body_images

Includes any images used in the main body of the article, requires “body” mixin.

attached_body_images

Attach any images from within the body of the article as <enclosure> tags (RSS 2.0).

clips

Includes any BBC hosted video or audio clips in the feed, see below sections on “Understanding the RSS 2.0 Response” for more detail on how this works.

This mixin requires “body” mixin to be present for text article feeds. But for media type (video and audio only) feeds “body” mixin is not required.

Media type feeds will contain "audio" or "video" in feed name. Any other feed should be considered as text article feed.

visual_journalism

Includes interactive elements in the body of articles (e.g. quizzes, expandable explainers). These appear as <iframe> tags within the HTML of the body content. Requires “body” mixin.

site_logo

Includes a logo for the BBC site the feed is coming from in the <channel> section, if available.

worldwide_verticals

Deprecated: Please do not use.

Includes articles that originally appeared on our ‘premium content’ websites, including BBC Future, BBC Earth, BBC Capital, BBC Culture and BBC Travel. Only to be used if your syndication deal includes this content, please ask your BBC Account Manager if you are not sure.

body_adverts

Includes advertisements within the body of an article, only supported on Apple News Format.

Understanding the RSS 2.0 Response

The Information Syndication API provides content as RSS 2.0 feeds. These are generated based on the top 10 articles currently present on the relevant BBC feed when you make the request for a feed.


Article Body

We include the main body of an article as a CDATA section within a <content:encoded> tag (using the RSS Content Module http://purl.org/rss/1.0/modules/content/).

Example

The <description> field of each item is used to store the summary of the article.


Article Body Images

Within the body of the article we include any images inside the HTML5 <figure> tags. The child <img> tag of the <figure> references the image itself, while the <figcaption> child tag contains the caption to go along with the image and the <footer> child tag gives the name of the copyright owner.

Example

Article Thumbnail Images

An article may have one or more thumbnail images associated with it. These are included as <media:thumbnail> tags inside <media:content> tags within each <item> in the feed. The <media:content> tags also contain copyright and alt text information.

Standard Thumbnail Images

This is the default format which is provided by excluding the “thumbnail_image_format” query parameter:

Example

It is strongly suggested that partners use the standard thumbnail image format so that additional information can be included with the thumbnail image. This includes the copyright information, description (alt-text), and title.


Image Only Thumbnails

However, if your system is unable to parse the standard thumbnail image format you can use the query string parameter “thumbnail_image_format=image_only”

This will remove the title, description, and copyright information from the thumbnail.

Article Body Video Clips

Articles will occasionally contain video clips, these are optional and included by using the “clips” mixin parameter. Our journalists always make sure that the text is always readable without the clips included, so you'd never get something like “as you can see in the below video” that wouldn't make sense if you weren't syndicating the clip.

There are two ways to include video on your site, either by taking the URL to the MP4 file and playing that in your player software, or embedding the BBC player as an iframe.

We prefer you not to ingest the video clip into your own system as we prefer all playouts to come directly to us - this allows us editorial control and means we're better able to manage taking down clips if there is ever a legal reason to do so.

You can control which you get by using the “clip_format” query string parameter:

  • “clip_format=link” (default) will give you the link to the MP4 file
  • “clip_format=embedded_player” will give you the iframe to the BBC player
LINK:

Within the body of the article we include any video clips inside the HTML5 <figure> tags. The child <video> tag of the <figure> references the clip itself, while the <figcaption> child tag contains the caption to go along with the clip and the <footer> child tag gives the name of the copyright owner. Other metadata about the clip is included as attributes on the <video> tag, including an id attribute which will let you uniquely identify that clip in the future, even if the URL has changed (this could happen if we have created a new version of the clip). The poster attribute contains the URL of the still image to show before the video is played. The controls attribute adds video controls, like play, pause, and volume.

Media Articles

Some feeds contain items which are primarily an audio or video clip with an optional accompaniment of text. We represent these as <media:content> items within the <item>. There is still a <content:encoded> node for the body text that accompanies the clip.

Example

If you pass the query string parameter “clip_format=embedded_player” then instead of getting a <media:content> node you'll see the <iframe> to the BBC player at the top of the <content:encoded> node. See “Article Body Video Clips” for more information about this parameter.

FAQs

What is a Feed?

A feed is the section of the page of the BBC website that you wish to syndicate. When you make a request for articles you need to let us know which feed is required.

For example if you wanted to syndicate content from the BBC England news site (https://www.bbc.co.uk/news/england/) you would pass the feed value of “news-england”, or if you were syndicating the Spanish language BBC Mundo's front page (https://www.bbc.com/mundo) you would pass “mundo-front-page”.

These feeds are not simply lists of the most recent stories, they are editorially curated to make sure that the news is delivered to the reader in the best way possible.


Which formats are supported?

The Information Syndication API syndicates articles in the RSS 2.0 format (with extensions). Other formats may be added in the future based on demand. To request the document pass the value “application/rss+xml” into the Accept HTTP header.


Why can't I see some images used on the BBC website?

The BBC doesn't own the copyright to all images we show on the website so depending on the licensing agreement in place we may not be able to syndicate some images, particularly larger ones.


Why are there articles on the BBC website that aren't in the feed?

At this time the Information Syndication API currently only syndicates standard news stories.

Other types of articles, like coverage of live events are not included yet. If they are included in the future they may be an additional cost involved in syndicating that type of content. Please speak to your Account Manager if this type of content is of interest to you.

It may also be that there is a copyright issue preventing us from syndicating a particular article, in which case they will also be excluded.


What is the ‘xtor’ query parameter? Why does it appear on BBC links?

‘xtor’ is the name of the query parameter used for referral tracking. It is used to monitor referrals back to BBC content from our partners.


Why is there a blank image at the bottom of articles?

This image is used for tracking when an article is visited. It won't be visible in the article but without it article interaction cannot be tracked.
